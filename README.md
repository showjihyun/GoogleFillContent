# FillContent

구글시트의 산출물 위치(구글 드라이브 링크)를 자동으로 채워주는 Google Apps Script 도구.

솔루션 상용화 표준 매뉴얼 등의 시트에서 항목/내용 컬럼을 읽고, 구글 드라이브 내 관련 파일을 검색하여 링크를 자동 기입합니다.

## 주요 기능

- **키워드 + AI 듀얼 매칭** — 키워드 기반 Drive fullText 검색 + Gemini AI 의미 기반 매칭
- **복수 검색 컬럼** — 여러 컬럼을 동시에 선택하여 검색 기준으로 사용 가능
- **버전 자동 판별** — 동일 파일의 여러 버전 중 최신 버전 자동 선택 (v1.0 < v2.0, draft < final)
- **자동 업데이트** — 매일 자동으로 산출물 변경(이동/삭제/새 버전)을 감지하여 경로 갱신
- **6분 제한 대응** — 시간 기반 자동 중단 + 이어서 실행으로 대량 항목도 처리 가능
- **매칭 로그** — 모든 매칭 활동을 숨김 시트에 기록하여 감사 추적 가능

## 설치 방법

### 1. Apps Script 프로젝트 생성

1. 대상 구글시트를 엽니다
2. **확장 프로그램** > **Apps Script** 클릭
3. 기본 생성된 `Code.gs`의 내용을 삭제합니다

### 2. 파일 추가

Apps Script 편집기에서 아래 파일들을 추가합니다.

**스크립트 파일 (.gs)** — 좌측 `+` 버튼 > `스크립트` 선택:

| 파일명 | 설명 |
|--------|------|
| `Code` | 진입점, 메뉴 등록, 사이드바 열기 |
| `StateManager` | 상태 관리, LockService, 트리거 |
| `VersionResolver` | 버전 판별, 기본명 추출 |
| `FolderTree` | 폴더 트리 재귀 수집, 경로 캐시 |
| `KeywordSearch` | Drive API 키워드 검색 |
| `AISearch` | Gemini API AI 검색 |
| `SearchEngine` | 메인 처리 루프 |
| `AutoUpdate` | 자동 업데이트 트리거 |
| `TestRunner` | 단위 테스트 (개발용) |

**HTML 파일** — 좌측 `+` 버튼 > `HTML` 선택:

| 파일명 | 설명 |
|--------|------|
| `Sidebar` | 사이드바 UI |

각 파일의 내용을 이 Repository의 해당 파일에서 복사하여 붙여넣습니다.

### 3. Drive API 활성화

1. Apps Script 편집기 좌측 메뉴에서 **서비스** (+ 아이콘) 클릭
2. **Drive API** 선택 > **추가** 클릭

### 4. Gemini API 키 설정 (AI 모드 사용 시)

1. [Google AI Studio](https://aistudio.google.com/apikey)에서 API 키 발급
2. Apps Script 편집기 > **프로젝트 설정** (톱니바퀴) > **스크립트 속성**
3. 속성 추가: 키 = `GEMINI_API_KEY`, 값 = 발급받은 API 키

> AI 모드를 사용하지 않는 경우 이 단계는 건너뛰어도 됩니다. 키워드 모드는 API 키 없이 동작합니다.

### 5. 실행

1. 구글시트를 **새로고침**합니다
2. 상단 메뉴에 **FillContent**가 표시됩니다
3. **FillContent** > **사이드바 열기** 클릭

> 첫 실행 시 권한 승인 팝업이 표시됩니다. "고급" > "안전하지 않은 페이지로 이동" > "허용"을 클릭하세요.

## 사용 방법

### 기본 사용

```
1. 사이드바 열기
   메뉴 > FillContent > 사이드바 열기

2. 검색 기준 컬럼 선택 (복수 선택 가능)
   매뉴얼의 "항목", "내용" 등 검색에 사용할 컬럼을 체크

3. 결과 기입 컬럼 선택
   검색된 파일 링크가 기입될 컬럼 선택

4. 비고 컬럼 선택 (선택사항)
   매칭 상태(성공/다중 후보/실패)가 기록될 컬럼

5. 시작 폴더 지정
   구글 드라이브에서 검색할 폴더의 URL을 붙여넣기
   (예: https://drive.google.com/drive/folders/1ABC...)

6. 매칭 모드 선택
   - 키워드: Drive fullText 기반 빠른 검색 (API 키 불필요)
   - AI: Gemini 기반 의미 검색 (API 키 필요, 더 정확)

7. "검색 시작" 클릭
```

### 매칭 모드

#### 키워드 모드
- 선택한 컬럼의 텍스트에서 키워드를 추출하여 Drive fullText 검색
- 파일 내용(Google Docs, Sheets, Slides, PDF)까지 검색
- 빠르고 예측 가능, API 키 불필요

#### AI 모드
- Gemini API로 매뉴얼 항목의 의미를 분석하여 핵심 키워드 추출
- 검색 결과 후보의 관련도를 0-100점으로 평가
- 70점 이상만 매칭, 다중 후보 시 최고점 자동 선택
- AI 실패 시 자동으로 키워드 모드로 fallback

### 버전 관리

동일 파일의 여러 버전이 있을 때 자동으로 최적 버전을 선택합니다:

| 우선순위 | 기준 | 예시 |
|---------|------|------|
| 1 | 버전 번호 최고 | `설계서_v2.0.docx` > `설계서_v1.0.docx` |
| 2 | final/approved > 일반 > draft/temp | `보고서_v1.0-final.docx` > `보고서_v1.0-draft.docx` |
| 3 | 수정일 최신 (버전 없는 경우) | 최근 수정된 파일 우선 |

인식하는 버전 패턴: `v1.0`, `V2.1`, `v1.0.3`, `1.0` 등

### 자동 업데이트

1. 메뉴 > **FillContent** > **자동 갱신 설정** 클릭
2. 매일 새벽 2시에 자동으로 실행되어:
   - 삭제된 파일 감지 → 셀 비우기 + 비고에 "파일 삭제됨" 기록
   - 이동된 파일 감지 → 경로 자동 갱신
   - 새 버전 감지 → 최신 버전으로 경로 교체

### 결과 확인

검색 완료 후 사이드바에 결과 요약이 표시됩니다:
- ✅ **매칭 성공** — 파일 링크가 결과 컬럼에 기입됨
- ⚠️ **다중 후보** — 최고 점수 파일이 기입되고, 비고에 다른 후보 목록 표시
- ❌ **실패** — 비고에 실패 사유 기록

### 매칭 로그

사이드바 하단 **"매칭 로그"** 버튼으로 숨김 시트(`_FillContent_Log`)를 열 수 있습니다.

모든 매칭 활동이 기록됩니다:
- 타임스탬프, 항목명, 검색 키워드
- 후보 파일 수, 선택된 파일, 선택 이유
- 매칭 모드, AI 점수

## 시트 구조 예시

| 항목 | 내용 | 산출물 위치 | 비고 |
|------|------|------------|------|
| 요구사항 정의 | SW 요구사항 정의서 작성 | (자동 기입) | (자동 기입) |
| 설계 | 시스템 아키텍처 설계서 | (자동 기입) | (자동 기입) |
| 테스트 | 통합 테스트 결과 보고서 | (자동 기입) | (자동 기입) |

- **항목 + 내용** → 검색 기준 컬럼으로 복수 선택
- **산출물 위치** → 결과 기입 컬럼
- **비고** → 비고 컬럼

## 제한사항

- Apps Script 실행 시간 제한: 6분/실행 (자동으로 이어서 실행하므로 사용에 영향 없음)
- Google Drive fullText 검색은 스캔된 이미지 PDF, 손글씨 등은 인덱싱하지 못할 수 있음
- 새로 업로드된 파일은 Drive 인덱싱에 수분~수시간 소요될 수 있음
- HWP 파일은 Drive fullText 검색 미지원
- 공유 드라이브(Shared Drive) 검색은 별도 권한 설정 필요

## 파일 구조

```
FillContent/
├── Code.gs              # 진입점, 메뉴, 사이드바, 서버-클라이언트 브릿지
├── SearchEngine.gs       # 메인 처리 루프, Strategy 패턴
├── KeywordSearch.gs      # Drive API fullText 키워드 검색
├── AISearch.gs           # Gemini API AI 검색
├── FolderTree.gs         # 폴더 트리 재귀 수집, 경로 캐시
├── VersionResolver.gs    # 버전 판별, 기본명 추출, 비교
├── AutoUpdate.gs         # 자동 업데이트 트리거
├── StateManager.gs       # 숨김 시트 상태 관리, LockService
├── TestRunner.gs         # 순수 로직 단위 테스트
└── Sidebar.html          # 사이드바 UI
```
